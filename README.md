# 青禾智学课后服务平台

**QINGHE After-school Service Platform v1.2**

青禾智学课后服务平台面向中小学课后服务场景，围绕课程开设、报名管理、上课记录、学习反馈、作业问答、家长陪学、亲子任务、积分兑换、平台公告、资源推荐与运营看板，构建覆盖“课程服务、学习过程、家校协同、平台治理”的一体化工作台。

## 产品定位

平台服务对象包括学校管理员、课后服务教师、学生和家长。系统以课程为核心，以学习记录和家校反馈为过程留痕，以真实 AI 辅助、家长陪学和亲子任务扩展服务边界，支持课后服务从报名、上课、反馈到家庭陪伴的闭环管理。

## 核心能力

- 平台首页：展示课程、问答、家长建议、亲子任务、活跃用户、近期动态和近期课程安排等全局运营数据。
- 课程中心：支持课程发布、唯一课程编号、开课范围、教师类型绑定、课程状态自动判定、报名状态筛选和上课提醒。
- 学生档案：维护学生基础信息、年级、家长绑定、兴趣标签与成长记录，为课程推荐和家校协同提供基础数据。
- 学习记录 / 上课记录：学生填写课中学习记录和结课总结，教师填写每次课反馈和结课反馈，家长与管理员按权限查看。
- 作业问答：学生提交学习问题，系统调用 AI 生成辅助解答，教师和管理员可查看问答记录与处理状态。
- 家长陪学：家长结合孩子情况生成陪伴建议，系统沉淀历史建议，并提供家长课堂资源。
- 亲子任务：家长发布家庭任务和积分奖励，学生提交完成证明，家长检查确认后发放积分并处理兑换申请。
- 网课中心：整合适合中小学生的拓展学习资源，支持按学习主题进行资源推荐。
- AI 中心：统一展示 AI 能力入口、模型配置、调用记录、返回内容和状态管理。
- 平台用户：管理学生、家长、教师、管理员账号，并支持教师教学类型的唯一绑定。
- 平台公告：发布学校通知、课程提醒和服务公告，支持已读记录跟踪。

## 角色功能

- 管理端：查看全平台数据，维护平台用户、课程资源、公告、AI 调用记录、家长陪学记录和亲子任务数据。
- 教师端：发布和维护本人课程，查看报名学生名单，填写上课记录、每次课反馈和结课反馈，处理作业问答。
- 家长端：查看绑定孩子档案、课程报名、学习记录和教师反馈，使用家长陪学建议，发布亲子任务并确认积分兑换。
- 学生端：查看可报名课程和已报课程，提交作业问题，填写学习记录和课程总结，完成亲子任务并发起奖励兑换。

## 业务流程

1. 管理员维护用户、基础资源和公告。
2. 教师根据教学类型发布课程，设置开课范围、上课周期、人数容量和上课地点。
3. 学生或家长在课程中心查看符合年级范围且无时间冲突的课程并完成报名。
4. 课程进行中，学生填写课中学习记录，教师填写本次课反馈。
5. 课程结课后，学生提交课程总结，教师提交结课反馈。
6. 家长通过学习记录、家长陪学建议和亲子任务持续跟进孩子课后成长。
7. 管理员通过平台首页、AI 中心和各业务模块掌握整体运行情况。

## 系统模块

```text
qinghe-admin        后端启动与接口控制模块
qinghe-common       通用工具、基础模型和公共封装
qinghe-framework    安全认证、权限控制和 Web 基础能力
qinghe-system       课后服务业务核心模块
qinghe-ui           前端交互界面
sql                 数据库初始化、升级和演示数据脚本
doc                 项目文档与补充材料
runtime             本地运行临时目录
```

## 技术组成

后端采用 Spring Boot、Spring Security、MyBatis、PageHelper、JWT、Druid、Fastjson2、MySQL 和 Maven，负责身份认证、权限控制、业务接口、数据访问、分页查询与系统配置。

前端采用 Vue、Vue Router、Vuex、Element UI、ECharts、Axios、Sass 和 SortableJS，负责分端路由、页面交互、状态管理、数据可视化和后台管理界面。

AI 能力由平台统一封装为业务服务，覆盖作业问答、课程推荐、家长陪学建议、课程通知和教学建议等场景，并通过 AI 中心沉淀调用记录与状态。项目支持在本地私有配置文件中按项目配置 API Key，该文件已被 Git 忽略，不会进入远端仓库。

## 后端启动

在项目根目录执行：

```powershell
mvn -U clean compile -DskipTests
```

启动类：

```text
qinghe-admin/src/main/java/com/qinghe/EduPlatformApplication.java
```

默认后端地址：

- [http://localhost:8080/](http://localhost:8080/)

## 前端启动

进入前端目录后执行：

```powershell
cd qinghe-ui
npm install
npm run dev
```

默认前端地址：

- [http://localhost/](http://localhost/)

## 数据库脚本

`sql` 目录包含基础表结构、平台业务表、升级脚本和演示数据。常用导入顺序：

1. `sql/qinghe_system_base.sql`
2. `sql/edu_after_school.sql`
3. `sql/edu_platform_bootstrap.sql`
4. `sql/edu_platform_upgrade.sql`
5. `sql/edu_demo_data.sql`
6. 根据展示需要继续导入扩展演示脚本

默认数据库名为 `qinghe_edu`，如需连接已有数据库，可通过环境变量 `MYSQL_DATABASE` 覆盖。
